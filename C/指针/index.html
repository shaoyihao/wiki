    指针

    指针变量¶

    指针变量存储某类型对象的内存地址，仅知道指针变量所存储的地址编号是不够的。在知道首地址之后，编译器还需要知道，从该地址开始，多少个字节内的数据是有效的，并且该用什么数据类型去理解这些数据？因此指针变量还需要指定其类型。因此任何指针变量都有两个属性：地址和类型。

    void*

    void 类型的指针只保存一个地址，不包含指向变量的类型信息，所以任何指针变量都可以直接赋值给它，无需进行强制类型转换，但是此时原指针变量的类型信息就丢失了，因此对 void 类型指针进行算术运算和取值操作都是不允许的。如果要将 void 类型的指针赋给其它类型的指针，则需要进行强制类型转换，使其包含类型（长度）信息。

    & 可用于取一个对象的地址，因此初始化可写为：

    1
    int *p = &x;  // *p是int型，也即 p 指向一个int型变量

    注意

    此处 * 是间接引用运算符，其优先级比算术运算符的优先级高。不过由于 * 和 ++ 都满足从右至左的结合顺序，因此 (*p)++ 中的括号不能省略，否则 *p++ 表达的含义将是 *(p++)。

    指针的算术运算¶

    若 p 是指向数组中某个元素的指针，那么 p++ 将指向下一个元素，p + i 将指向其后第 \(i\) 个元素，p - i 指向前面第 \(i\) 个元素。对指针变量进行算术运算所表达的含义为指针的“偏移”。实际地址的改变量为指针变量的类型的大小 × \(i\)，不过该转化过程对程序员是透明的。

    此外，指针（地址）之间还可以进行比较（包括 ==、!=、>=、< 等），例如若指针 p 和 q 指向同一个数组，如果 p 指向的数组元素的位置在 q 指向的数组元素位置之前，则 p < q 为真。此时 q - p + 1 就是位于 p 和 q 指向的元索之间的元素的数目。

    指针与整数之间不能相互转换，也不能把一个数值赋给指针变量，但 0 是例外：常量 0 可以赋值给指针，指针也可以和 0 进行比较。程序中经常用符号常量 NULL 代替常量 0。任何指针与 0 进行相等或不等的比较运算都有意义。

    指针与数组¶

    对于数组元素 a[i]，C 语言实际上最终还是将其转换为 *(a+i) 的形式，因此在程序中这两种形式是等价的。更进一步，&a[i] 与 a+i 的含义也是相同的，是相应数组元素的地址。a 其实是一个指针常量，值为数组第一个元索的地址。
