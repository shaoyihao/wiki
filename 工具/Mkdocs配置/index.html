    MkDocs的使用¶

    MkDocs（Markdown Documents）是一个快速、简单的静态网站生成器。文档源文件采用 Markdown 编写，且只需单个 YAML 配置文件进行配置，另外还有第三方主题和插件可以进行扩展。

    安装¶

    Bash
    1
    pip install mkdocs

    基本使用¶

    Bash
    1
2
3
4
5
6
    mkdocs new myProject      # 创建一个Mkdocs项目
cd myProject
# 另一种方法：
# mkdir myProject 
# cd myProject
# mkdocs new .

    MkDocs 带有一个内置的开发服务器，可以实现在处理文档时实时预览文档，并且还支持自动重新加载，并且只要配置文件、文档目录或主题目录中的任何内容发生更改，就会重新构建文档。

    Bash
    1
    mkdocs serve              # 本地测试

    若要使用 material 主题，需：

    1. 下载该主题

      Bash
      1
      pip install mkdocs-material

    2. 修改配置文件 mkdocs.yaml：

      YAML
      1
2
3
      site_name: XXX
theme:
  name: material

    Mkdocs 项目文件夹下的 docs 中存放着所有 Markdown 文件，我们也可根据自己的需要进一步用文件夹归类划分。docs 是 MkDocs 运行时默认的根路径，该目录下必须要有一个 index.md 文件，它是整个 MkDocs 构建的首页入口。

    转化为静态网页项目¶

    Bash
    1
    mkdocs build

    执行该命令后将生成一个 site 文件夹，其中即包含了整个网站的文件（静态 HTML 文件、CSS、JavaScript 和其他资源），将该文件夹下的内容上传至 github、云服务器、...等即可实现部署。

    部署至github page¶

    若要部署到 github page，Mkdocs 提供了简便的方法。

    基本步骤为：

    1. 创建一个 github repository

    2. 将该项目（此处以wiki为例） clone 到本地：

      Bash
      1
      git clone https://github.com/shaoyihao/wiki.git

    3. 以该文件夹作为Mkdocs项目

      Bash
      1
2
      cd wiki
mkdocs new .

    4. 一键部署到 github page：

      Bash
      1
      mkdocs gh-deploy

      注意到，这里与我们前文所提的“将 site 文件夹”上传到远端以实现部署不同。此处根本无需执行 mkdocs build，项目中也根本没有 site 文件夹，直接执行 mkdocs gh-deploy 即可。

      但通过查看 github 中所存储的文件可以发现，github 上所存的正是 site 中的文件！ How？

      原来首次执行 mkdocs gh-deploy 时会：

      • 创建 site （即隐含执行了 mkdocs build）
      • 创建 git 分支 gh-pages

      之后会将生成的文件使用 ghp-import 工具提交到gh-pages分支，然后将该分支推送到 GitHub。

    一般做法¶

    在本地进行文件的编辑、实时预览，最终再将文件推送到 github。在推送时无需将 site 文件夹也提交，因此可将其添加到 .gitignore 中。

    Bash
    1
    echo "site/" >> .gitignore

    文件被删除后仍将驻留在 site 中，要删除那些陈旧的文件，只需在执行命令时带上 --clean 参数即可。

    Bash
    1
    mkdocs build --clean

    markdown 扩展¶

    MkDocs 使用 Python-Markdown 库（Markdown 规范的 Python 实现）来渲染 Markdown 内容，因此我们可以在 Python-Markdown 的 官方文档 中浏览到目前所支持的 Markdown 扩展有哪些。

    Bash
    1
    pip install pymdown-extensions

    数学公式¶

    YAML
    1
2
3
4
    markdown_extensions:
  - pymdownx.arithmatex
extra_javascript:
  - 'https://cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.0/MathJax.js?config=TeX-MML-AM_CHTML'
