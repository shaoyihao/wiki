    Git 是目前世界上最先进的分布式版本控制系统，由 Linus 于 2005 年用 C 语言开发，在开发中具有广泛的应用，因此学习 Git 是十分必要的。

    版本控制¶

    版本控制系统（VCS）是一类用于追踪项目文件夹改动的工具，同时也使团队协作变得更加方便。

    常见的版本控制系统可以分为集中式与分布式两类。

    • 集中式版本控制系统：版本库集中存放于中央服务器，工作时首先从中央服务器获取最新版本，工作完再将成果推送至中央服务器。
    • 分布式版本控制系统：每个人的电脑上都是一个完整的版本库，多人协作时只需把各自的修改推送给对方，就可以互相看到对方的修改了。系统中通常也有一台充当“中央服务器”的电脑，仅用于方便“交换”大家的修改。

    所有的版本控制系统，其实只能跟踪文本文件的改动，比如 txt 文件、程序代码等等，Git 也不例外。版本控制系统可以告诉你每次的改动，比如在第 5 行加了一个单词 “Linux”、在第 8 行删了一个单词 “Windows” 等。而图片、视频这些二进制文件，虽然也能由版本控制系统管理，但没法跟踪文件的变化，只能把二进制文件每次改动串起来，也就是只知道图片从 100KB 改成了 120KB，但到底改了啥，版本控制系统无法知道。

    注意

    注意：Microsoft 的 Word 格式是二进制格式，因此版本控制系统是没法跟踪 Word 文件的改动的。

    如果要使用版本控制系统，就要以纯文本方式编写文件。

    Git 的安装与基本配置¶

    可在 Git 官网下载，不过对于 macOS 系统，Apple 开发者网站中下载的命令行工具中包含了 Git。

    设置用户信息：

    1
2
3
    git config --global user.name "Your Name"
git config --global user.email "email@example.com"
# 这些信息将存储于 `~/.gitconfig`

    注意

    如果要在某项目中使用其他名字或者电邮，上述命令去掉 --global 选项，该配置将保存在当前项目的 .git/config 文件里。

    查看已有的配置信息，可以使用 git config --list 命令。
