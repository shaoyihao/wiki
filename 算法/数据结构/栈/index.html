    基本理论

    栈（stack）是只允许在一端（栈顶）进行插入、删除元素的（受限）线性表。特点： 后进先出（LIFO）。

    顺序存储结构——顺序栈¶

    用一组地址连续的存储单元存储数据，通过 top 指向（记录）当前栈顶元素的位置。

    #define MAXSIZE 50
typedef struct {
    ElemType data[MAXSIZE];
    int top; // 指向栈顶元素
} SeqStack;

      void Pop(SeqStack &S, ElemType &e)
{
    if (S.top == -1) return; // 栈空
    e = S.data[S.top];
    S.top--;
}

      因为顺序存储删除一个元素时并没有销毁该空间，所以出栈的元素其实还在那个位置，只不过下次再有元素进栈时，就把它覆盖了。

    共享栈¶

    利用栈底位置相对不变的特性，可让 2 个顺序栈共享一个一维数组空间：将 2 个栈的栈底分别设置在数组的 2 端，栈顶向数组中间延伸，当两个栈顶相邻时，栈满。

    1
2
3
4
    typedef struct {
    ElemType data[STACKSIZE];
    int top1, top2; // 两个栈各自的栈顶 
} SSeqStack;

    链式存储结构——链栈¶

    用单链表实现栈时一般没有头结点，首元结点就是栈顶元素，所有操作都在表头进行。

    1
2
3
4
5
6
    typedef struct Node 
{
    ElemType data;
    struct Node *next;
} Node;
typedef Node* LinkStack;

    基本操作：

    • 初始化

2
3

{
    top = NULL;
}

      判断栈空

{
    return (top == NULL);
}

      获取栈顶元素

      1

    else return top->data;
}

}

      void DestroyStack(LinkStack top)

    StackNode *p = top;

        delete tmp;

}

}

      void Pop(LinkStack &top, ElemType &e)
{
    if (top == NULL) return ERROR; // 栈空
    e = top->data;

    Node *tmp = top;
    top = top->next; // 更新top
    delete tmp;
}

    顺序栈和链栈在时间效率上差不多；而在空间效率方面，顺序栈需要预先分配固定长度的空间，可能造成空间浪费或溢出，链栈一般不会出现溢出，但是每个结点需要额外的指针域，开销增加。因此如果元素个数变化较大，可以采用链栈；反之可以采用顺序栈。
