    逆序数

    题目：给定一个长度为 \(n\)​ 的整数序列，求逆序数。

    思路1：每次交换相邻元素¶

    由于交换 2 个相邻元素最多使逆序对数目 - 1，因此发生交换的总次数等于该数组中的 逆序对的数目。

    时间复杂度：\(O(n^2)\)

    long long inv(int q[], int n)
{
    long long cnt = 0;
    for (int i = 0; i < n - 1; i++)         // 冒泡排序
        for (int j = 0; j < n - i; j++)
            if (q[j] > q[j + 1])
            {
                swap(q[j], q[j + 1]);
                cnt++;
            } 
    return cnt;
}

    思路2：分治¶

    将数列 \([l,r]\) 分为两部分 \([l,\text{mid}],[\text{mid}+1,r]\)，则逆序对可以分为 3 类：

    • 两个数都在左部分
    • 两个数都在右部分
    • 一个数在左部分，一个数在右部分

    设函数 f(l, r) 可以将序列排序并返回该序列的逆序数，则：

    \[ f(l,r)=f(l,\text{mid})+f(\text{mid}+1,r)+f_3 \]

    对于第 ③ 类情况，在执行完 f(l, mid) 与 f(mid+1, r) 后，序列 \([l,\ \text{mid}],[\text{mid}+1,\ r]\) 有序。子列 2 的元素下标均大于子列 1 的下标，因此对于子列 2 中的元素 \(q_j\)，若子列 1 中有 \(s_j\) 个元素大于 \(q_j\)，则含 \(q_j\) 的逆序对的数目即 \(s_j\)。从而第 ③ 类情况的逆序数 = \(s_1+s_2+\cdots+s_m\)（\(m\) 为子列 2 的元素数目）。

    不过对于子列 2 中的 \(q_j\)，如何快速判断子列 1 中有多少个元素比它大呢？

    考虑到在归并排序的“归并”操作中，较小的数不断被取出，先被 \(i,j\) 指针指到的元素均小于后面指到的元素：

    • \(i\) 指针之前的元素均小于 \(q[i]\)
    • 若 \(q[i]>q[j]\)，则 \(q[i\sim \text{mid}]\) 均大于 \(q[j]\)

    因此子列 1 中比 \(q_j\) 大的元素个数为 \(\text{mid}-i+1\)，也就是含 \(q_j\)​​ 的逆序对的数目。

    时间复杂度：\(O(n\log n)\)

    long long inv(int q[], int l, int r)
{
    if (l >= r) return 0; // 当数列中只有一个元素时，逆序数就是0

    int mid = (l + r) / 2;
    long long res = inv(q, l, mid) + inv(q, mid + 1, r); // 第1类和第2类的逆序数

    int *tmp = new int[r - l + 1];
    int i = l, j = mid + 1, k = 0;
    while (i <= mid && j <= r)
    {
        if (q[i] <= q[j]) tmp[k++] = q[i++];
        else  // 此时为q[i]>q[j]的情况，即存在第3类逆序数的情况
        {
            tmp[k++] = q[j++];
            res += mid - i + 1; // 加上第3类情况的逆序数
        }
    }
    while (i <= mid) tmp[k++] = q[i++];
    while (j <=  r ) tmp[k++] = q[j++];
    for (int i = l, k = 0; i <= r; i++, k++) q[i] = tmp[k];

    delete [] tmp;
    return res;
}
