    第 k 小元素

    题目：给定一个长度为 \(n\) 的整数数列，求其中第 k 小的数。

    解法1¶

    受快速排序中 Partition 的启发，数列 \(q[l,r]\) 被分成 3 部分：\(q[l,\ t-1]\le q[t]\le q[t+1,\ r]\)。\(q[l,t]\) 包含的元素数目 \(s=t-l+1\)，且 \(q[t]\) 即第 \(s\) 小的数。

    • 若 \(k=s\)，则返回 \(q[t]\)；
    • 若 \(k<s\)，则只需在 \(q[l,\ t-1]\) 中找第 \(k\) 小的数；
    • 若 \(k>s\)，则只需在 \(q[t+1\ ,r]\) 中找第 \(k-s\) 小的数。

    int f(int q[], int l, int r, int k) // 0 < k <= r-l+1
{
    int t = partition(q, l, r); // t∈[l,r]
    int s = t - l + 1; 
    if (k == s) return q[t];
    else if (k < s) return f(q, l, t - 1, k);
    else return f(q, t + 1, r, k - s);
}

    该算法最坏情况下的时间复杂度为 \(O(n^2)\)，不过平均时间复杂度仅为 \(O(n)\) 。

    解法2¶

    在快速排序中，令每个元素起始时（乱序）时的下标为 \(a_i\)，有序时的下标为 \(A_i\)，由于快排是一种原址排序，整个排序过程可以视为一个数组在原地不断进行元素之间的交换，使得整体上每个元素越来越接近它最终的位置 \(A_i\)。 每一层递归实际上就是使得部分元素离最终位置更近一分（总有主元落到最终位置），不妨称这个过程为“归位”。

    因此要找第 \(k\) 小的数，也就是找最终会落在下标 \(k-1\) 的那个元素，我们每次只需对包含下标 \(k-1\) 的区间进行“归位”。

    int f(int q[], int l, int r)
{
    int t = partition(q, l, r);
    if (t == k - 1) return q[t];
    else if (t < k - 1) return f(q, t + 1, r);
    else return f(q, l, t - 1);
}
