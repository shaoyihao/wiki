    质数

    质数只定义在大于等于 \(2\) 的数中，小于等于 \(1\) 的数既不是质数，也不是合数。

    质数的判断¶

    暴力试除¶

    时间复杂度：\(\Theta(n)\)

    bool isPrime(int x)
{
    if (x < 2) return false;

    for (int i = 2; i < x; i++)
        if (x % i == 0) return false;

    return true;
}

    优化¶

    定理

    若 \(d\ |\ n\)，则 \(\dfrac{n}{d}\ |\ n\)。

    该结论揭示了：\(n\) 的因数都是成对出现的（\(d,\dfrac{n}{d}\)），因此我们在枚举时只需要枚举每一对因数中较小的那一个，范围为 \([2,\sqrt n]\)，改进后的算法时间复杂度为 \(\Theta(\sqrt n)\)。

    bool isPrime(int x)
{
    if (x < 2) return false;

    for (int i = 2; i <= x / i; i++)
        if (x % i == 0) return false;

    return true;
}
// for (int i = 2; i <= sqrt(x); i++) 不推荐，sqrt函数较慢
// for (int i = 2; i * i <= x; i++) 不推荐，存在i*i溢出风险

    质数的筛选¶

    筛质数的基本思路都是：将合数筛去，留下的都是质数。

    朴素筛¶

    利用的原理是 任意整数 \(x\) 的倍数 \(2x,3x,…\) 等都不是质数 。从小到大进行枚举，每枚举一个数，就将整个数列中该数的倍数筛去，最终剩下的数就是质数。

    时间复杂度：\(O(n\log n)\)

    bool st[N];           // 记录该数是否被筛除
int primes[N], cnt;   // cnt记录所筛出质数的数目

void get_prime(int n) // 将小于等于n的所有质数都筛选出
{
    for (int i = 2; i <= n; i++) // 枚举所有数
    {
        if (!st[i]) primes[cnt++] = i; // 若该数没被筛除，说明前面所有数均不是它的因数，即该数的因数只有1和它本身，所以i为质数

        for (int j = i * 2; j <= n; j += i) st[j] = true; // 将i的2,3,4,...倍数都筛除
    }
}

    注意到：此处对于每一个数，我们都枚举一遍它的倍数，将其筛除，如此存在着许多重复筛除的情况，降低了算法效率。

    埃氏筛¶

    由于合数都是质数的倍数，因此我们只需要从小到大枚举质数时，将该质数的所有倍数删除即可。

    时间复杂度为 \(O(n \log\log n)\)，近似为 \(O(n)\)。

    void get_prime(int n)
{
    for (int i = 2; i <= n; i++) 
        if (!st[i]) // 若该数没被筛除，说明该数为质数
        {
            primes[cnt++] = i; 
            for (int j = i * 2; j <= n; j += i) st[j] = true; // 将该数的所有倍数都筛除
        }
}

    注意到，埃氏筛法中依旧存在重复标记的现象，例如 12 既会被 2 又会被 3 标记：在标记 2 的倍数时，12=6∗2，在标记 3 的倍数时，12=4∗3 。根本原因是没有找到唯一产生 12 的方式。

    线性筛¶

    基本思路与埃氏筛相同：将每个合数借助其某个质因子而筛去。

    定理

    任意一个大于 \(1\) 的数都满足：\(数=最小质因数\times最大真因数\)。

    若 \(p\) 是一个 小于等于 \(x\) 最小质因数 的质数，则 \(px\) 就是一个最大真因数为 \(x\) 的合数。

    已知一个数 \(x\)（\(x>1\)），我们可以得到所有小于等于 \(x\) 的最小质因数 \(t\) 的质数 \(p_1,p_2,...,p_k\)，那么就可以筛去 \(p_1x,p_2x,...,p_kx\)（所有以 \(x\) ​为最大真因数的合数）。

    void get_prime(int n)
{
    for (int i = 2; i <= n; i++)
    {
        if (!st[i]) prime[cnt++] = i; // 此时prime数组中存的是所有小于等于i的质数
        for (int j = 0; prime[j] * i <= n; j++) // 从小到大枚举质数（写成prime[j] <= n / i 更好）
        { // 借助小于等于i的最小质因数的质数们 筛去 以i为最大真因数的合数
            st[prime[j] * i] = true; // prime[j]是小于等于 i最小质因数 的质数，则 prime[j]*i是以i为最大真因数的合数，筛去
            if (i % prime[j] == 0) break; // 此时，prime[j] 一定是 i 的最小质因数（从较小的质数开始枚举的），此时所有以i为最大真因数的合数都被筛去了，退出循环
        }
    }
}

    注意到：每个合数都会且只会被其最大真因数筛掉。又每个合数的最大真因数是唯一的，从而每个合数都只会被筛一次，因此线性筛法可以使得：每个数只需要被遍历一次就可以判断是否筛去（即质数还是合数），时间复杂度为：\(O(n)\)​ 。

    一些问题

    • 为什么每个合数都会被筛掉？

      对于任意一个合数\(N\)，可以分解为：\(N=p\times x\)，其中\(p\)是\(N\)的最小质因数。显然\(p\)也小于等于\(x\)的最小质因数，因此当循环到\(x\)时，一定会枚举到质数\(p\)，此时就会筛掉\(px\)，即筛掉\(N\)。

    • 注意到内层for循环的判断条件为prime[j] <= n / i，理由是：

      该层for循环的作用是借助小于等于\(i\)的最小质因数的质数们( prime[j] )筛去以\(i\)为最大真因数的合数 prime[j]×i 。

      由于我们只需要筛出\([2\sim n]\)之间的质数，因此所要筛去的合数范围为prime[j]×i ≤ n

    • 为什么判断条件不是j < cnt && prime[j] <= n / i ？

      因为：若\(i\)是合数，\(i\)的最小质因数一定已经存于prime数组中了；若\(i\)是质数，则先直接加入prime数组中。因此prime数组中总会存在\(i\)的最小质因数，循环总是会在j≥cnt之前退出，所以不需要再加上这个条件。

    注意到：线性筛法还有一个功能就是求出每个数 \(x\)（\(x>1\)​）的最小质因数 minp[x]。

    void get_primes(int n)
{
    for (int i = 2; i <= n; i++)
    {
        if (!st[i]) // i是质数
        {
            minp[i] = i; // 质数的最小质因子即其本身
            primes[cnt ++] = i;
        }
        for (int j = 0; primes[j] * i <= n; j++) // 根据i筛合数
        {
            st[primes[j] * i] = true;
            minp[primes[j] * i] = primes[j]; // 此时primes[j]小于等于i的最小质因数，因此primes[j]即 合数primes[j]*i 的最小质因数
            if (i % primes[j] == 0) break;
        }
    }
}

    分解质因数¶

    合数一定可以分解为质数的乘积。

    暴力¶

    从小到大枚举所有数，若数 \(i\) 是 \(x\) 的因数，由于是从小到大枚举，因此此时 \(i\) 也一定是质数，即此时 \(i\) 即为 \(x\) 的一个质因数，再求出 \(x\) 中因数 \(i\) 存在多少次。

    void divide(int x)
{
    for (int i = 2; i <= x; i++)
        if (x % i == 0) // 若该判断条件成立，则i一定为质数
        {
            int cnt = 0;
            while (x % i == 0)
            {
                x /= i;
                cnt++;
            }
            printf("%d*%d ", i, cnt);
        }
}

    优化¶

    定理

    \(n\)中最多只包含一个大于\(\sqrt n\)的质因子。

    因此我们在枚举时可以先把所有小于等于 \(\sqrt n\) 的质因子枚举出来，最终剩余的 \(x\) 如果是一个大于 1 的数，那么此时 \(x\) 就是最后一个质因子。

    优化后的时间复杂度为\(O(\sqrt n)\)​ 。

    void divide(int x)
{ 
    for (int i = 2; i <= x / i; i++) //
        if (x % i == 0)
        {
            int cnt = 0;
            while (x % i == 0)
            {
                x /= i;
                cnt ++;
            }
            printf("%d %d\n", i, cnt);
        }

    if (x > 1) printf("%d 1\n", x); // 特判，是否存在大于根号x的质因子
}

    unordered_map<int, int> prime;
void divide(int x)
{
    for (int i = 2; i <= x / i; i++)
        while (x % i == 0) 
        {
            prime[i]++;
            x /= i;
        }
    if (x > 1) prime[x] = 1;
}
