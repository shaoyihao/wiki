    概述

    在有序数组中，通过二分法可以以 \(O(\log n)\)​​​ 快速查找元素。对于某些问题，若答案落在某个区间 \([l,r]\)​​​ 中，可以利用二分法对其进行搜索。

    将这类问题建模为一组对立命题 \(P,Q\)，将搜索区间划分为 2 部分（不妨称为左区间和右区间），取值分别满足 \(P,Q\)，而我们所要查找的 \(x\)​ 就是这 2 个区间的分界点。不断折半缩小查找的区间，缩小区间的依据即该数所满足的命题。

    关键思路在于弄明白：

    • 要查找的对象是什么
    • 查找范围是什么
    • 查找对象的所满足的关系式是什么（明确 check(mid) 函数什么时候为真，什么时候为假）

    注意

    二分的区间只要包含正确答案即可，即：所二分的区间不是唯一的（大一点较稳妥），根据题目而定。例如求 \(x\) 的平方根时，就要保证二分区间的右端点不小于 \(x+1\)。因为当 \(x = 0.01\) 时，开方后为 \(0.1\)，会超出了区间 \([ 0, x ]\)。

    因此，二分算法中很重要的一步就是：估计答案的范围。

    整数二分¶

    对于整数二分，左右区间是不相交的，

    可以分为 2 种情况：

    • 所要查找的 \(x\)​ 是左区间的右端点

      int l, r, mid;
while (l < r)
{
  mid = (l + r + 1) / 2;
  if (check(mid)) l = mid; // l = mid与mid = (l + r + 1) / 2 相对应
  else r = mid - 1;
}

    • 所要查找的 \(x\)​​ 是右区间的左端点

      1
2
3
4
5

7
      int l, r, mid;
while (l < r)
{
    mid = (l + r) / 2;
    if (check(mid)) r = mid; // r = mid与mid = (l + r) / 2 相对应
    else l = mid + 1;  
}

    上述二分过程最终一定会找到一个 \(x\)。若原数组中本就不存在所要查找的满足命题 \(p\) 的元素，则二分找到的元素就是距离最近的、满足 \(\bar{p}\)​ 的元素。

    例子

    在有序整数数组 \(q\) 中，若要查找 \(x\)，可设命题 \(P: q[i]\le x\)，\(Q:q[i]>x\)，从而所要查找的就是“左区间的右端点”。

    浮点数二分¶

    浮点数二分就不存在边界的问题了，因此简单许多。

    int l, r, mid;
while (r - l > 1e-8) // 注意循环结束的条件，区间很小时，视作l==r
{
    mid = (l + r) / 2;
    if (check(mid)) l = mid;
    else r = mid;
}
