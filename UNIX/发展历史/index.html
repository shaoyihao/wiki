    发展历史

    初代 Unix¶

    1964年 Bell 实验室（隶属于 AT&T 公司）、MIT、GE 的一些研究人员合作开发 Multics 操作系统。由于该项目过于繁杂，短期内难以实现，Bell 实验室于 1969 年退出了该项目，撤回了研究人员。这些研究员中的 Ken Thompson 等人从 Multics 中获取了灵感，他们决定创建自己的小型操作系统。

    特别是 Thompson 希望能够在自己的系统上运行他在 Multics 上开发的游戏 Space Travel。

    1969 年在 Bell 实验室中，Thompson 等人为 PDP-7 计算机开发了一个小型操作系统（用的是PDP-7 的汇编语言），并取名为 Unics（与 Multics 形成对比），后来改名为 Unix。C 语言发明后，又用 C 语言重写了 Unix。

    进一步发展¶

    1969~1979 年间，Unix 历经了多次版本迭代：

    版本 发布时间 特点
    1 1971.11 许多沿用至今的程序都已有了雏形，例如 cat、cp、mv 等
    2 1972.6
    3 1973.2 包括了 C 编译器，实现了管道
    4 1973.11 几乎完全以 C 语言重写
    5 1974.6
    6 1975.3

    BSD¶

    当时在美国政府的授权下，AT&T 垄断着全美电信市场，但 AT&T 公司与美国政府达成的协议条款禁止 AT&T 涉足软件销售行业，这意味着 AT&T 不能将 Unix 作为产品销售。因此 AT&T 索性准许高校在支付象征性的发布费用后使用 Unix 系统，还包括了相关文档及内核源码，这极大促进了 Unix 的普及和使用。

    此时 Unix 有两大开发基地，一个是 Bell 实验室，另一个是 UC Berkeley 的计算机科学技术系。UCB 的研究生 Bill Joy 独自开发了大量重要的软件，包括 vi 编辑器、C-Shell 等。1977 年第一版 Berkeley Unix 发布，称为 Berkeley 软件套件（Berkley Software Distribution，BSD），1978 年又发行了第二版：2BSD，1979 年 12 月诞生了首个完整的 Unix 发布版 3BSD。

    美国东海岸的 Unix（AT&T公司的UNIX）和西海岸的 Unix（BSD）一直平分秋色，且都发展很快，成为 Unix 的主要的 2 个流派，其它形式的 Unix 系统都是基于这 2 者的。

    随着 AT&T 在电话系统市场垄断地位的丧失，AT&T 也因而获准销售 UNIX。1979年，AT&T 公司宣布将 UNIX 系统作为商业产品进行销售，第一个商业版本的 Unix 称为 UNIX System III（1982年发布），很快 1983 年又发布了 UNIX System V。同时由于版权，所有的 BSD 用户也被要求付费购买 AT&T 公司的许可证。BSD 用户们决定重新编写 BSD 中基于 AT&T 公司 UNIX 的那一部分代码，直到 1992 年这一任务才彻底完成。

    Minix¶

    将 UNIX 商业化的同时，AT&T公司也不再允许其公司之外的人查看 UNIX 的源代码，这给大学里操作系统课程的教学带来了麻烦。为有一个免费、直接可用的、适合于教学的、可以查看源代码的操作系统，阿姆斯特丹 Vrije Universiteit 大学的 Andrew Tanenbaum 教授开发了自己的操作系统，称其为 Minix，于1987年发行。该系统主要用于教学目的，是一种很好的教学工具，但并不适合实际开发，因此大量程序员希望对其进行增强，但 Tanenbaum 坚持 Minix 应该是一个简单的操作系统，适合于教学需要，否决了大部分的请求。

    GNU¶

    为倡导开源共享，Stallman等人于1985年成立了自由软件基金会（Free Software Foundation，FSF）。Stallman 为 FSF 制定的首个目标就是创建一个可以自由共享的操作系统，为了使新操作系统可以自由发行，Stallman意识到操作系统必须从头开始编写。同时他希望 FSF 的产品能够平稳地融入到目前流行的编程文化中，因此他决定新的操作系统应该与 Unix 兼容，也就是说它看上去就像 Unix 系统一样，并且还能够运行Unix 程序，他给这一操作系统命名为 GNU（GNU's Not Unix）。

    在 20 世纪 80 年代后期，Stallman 在创建自由软件方面已经积累了一定的经验，他认为如果要创建一个大型的自由软件，那么就需要一个合适的许可证协议，并在这个许可证协议之下发行该软件。为了这个目的，他提出了非盈利版权（copyleft）的理念。传统的版权用来限制软件的使用，而非盈利版权则不同，以 Stallman 的话来说：“非盈利版权的中心思想就是授予任何人运行程序、复制程序、修改程序、发行修改后程序的权限，但是不能在自己修改后的软件上添加限制”。为了实现该理念，Stallman 提出并编写了通用公共许可证（general public license，GPL），并于1989年发布，之后开源活动便进入了空前繁荣的发展阶段。

    到1991年，Stallman 等人已经创建了大量的开放源代码软件，包括 Emacs、gcc、g++、gdb、Bash等实用工具。这些软件都是 GNU 项目的一部分，几乎构成了一套完整的操作系统，但他们还没有一个成型的 Unix 内核（Stallman称之为 HURD）。

    总之，在 1991 年这个时间点，AT&T 公司商业化了 UNIX、BSD 受到 AT&T 公司许可证协议的制约、Minix 不够完美且无法自由共享、GNU内核遥遥无期.....，人们渴望着拥有开放源代码的类Unix操作系统。

    Linux¶

    1991年，Linus Torvalds 是 Felsinki 大学计算机科学系的大二学生，他阅读了 Andrew Tanenbaum 的书 Operation Systems: Design and Implementation，该书解释了 Minix 的设计原则，并在书后附了 12000 行源代码。

    Linus 构建自己的操作系统纯属兴趣，因此他意识到自己只要编写内核就好了，之后他可以直接使用 FSF 提供的各种实用工具。1991年9月，Linus 通过因特网发行了第一版的内核，并命名为 Linux，之后又不断地进行更新，越来越多的人加入其中，如今 Linux 已成为世界上最流行的操作系统。

    严格地讲，Linus Torvalds 的 Linux项目只是一个内核，并不是一个完整的操作系统（而且现在情况依然如此）。当Linux 内核最初发布时，为了使用它，必须为它寻找所需的各种组件，并将它们组装在一起才能形成一个操作系统。在Linus 第一版内核发布数月之后，其他人就开始提供基于 Linux 内核的操作系统，一般称为 Linux 发行版（Linux distribution，有时候简写为distro）。其中 Patrick Volkerding 于1993年7月发行的 Slackware 十分成功，现在它是最古老的 Linux发行版，且仍然在积极地发展。

    现代的 Linux 发行版都提供：内核、实用工具、编程工具以及至少一个GUI。

    大多数 Linux 发行版都使用 FSF 的 GNU 实用工具，因此 FSF 坚持基于 Linux 的操作系统应称为 GNU/Linux。

    目前最流行的 Linux发行版有：Ubuntu、Red Hat、Slackware、Debian、Fedora Core等。

    BSD发行版¶

    1992年， 一个名叫 Bill Jolitz 的程序员完成了 BSD 代码重写的最后工作，创建了一个新版本的 BSD，称为 386/BSD，通过因特网发行。

    386/BSD 是 Unix 发展历史的一个里程碑。到此时，BSD 终于可以在全球作为开放源代码软件自由发行。

    很快 386/BSD 就流行起来，但 Jolitz 并没有充足时间进行维护，因此一组志愿者接管了这项工作。该小组所做的第一件事就是将该操作系统重新命名为 FreeBSD。FreeBSD 只能运行在 PC 硬件上。但是也有一些用户希望在其他类型的机器上运行BSD，因此成立了一个新的小组，该该小组的目标就是尽可能地将 FreeBSD 移植到其他类型的机器上，最终的成果为 NetBSD。后来又分出另一个小组，该小组主要关注安全和密码学问题，这个小组的成果为 OpenBSD。

    这就是 BSD 主要的 3 个发行版：FreeBSD、NetBSD、OpenBSD。
